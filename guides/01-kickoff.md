# キックオフ資料の作り方

> PJT-YPで確立したキックオフ資料の作成フロー・知見をまとめる。

---

## 全体フロー

```
① インプット収集
   ├─ 受領資料（提案書、既存システム資料、組織図等）
   ├─ 事前商談の文字起こし
   └─ ヒアリングメモ
       │
       ▼
② convert-to-text.mjs でテキスト化
   └─ xlsx/pptx/pdf → txt 一括変換
       │
       ▼
③ キックオフMD（templates/kickoff-format.md）に流し込み
   └─ 6セクション構造に沿って記述
       │
       ▼
④ 関係者とすり合わせ
   └─ MDベースで認識合わせ。特にセクション2「弊社理解」が核
       │
       ▼
⑤ 合意後にPPTX化
   └─ Notionに展開 → PPTXスライド化、または直接PPTX作成
       │
       ▼
⑥ 合意後は変更しない
   └─ 変更が必要な場合は定例で合意の上、版を上げる
```

---

## ステップ詳細

### ① インプット収集

プロジェクト開始前に入手可能な資料を全て集める:

- **提案書・契約書**: スコープ・期間・費用の前提
- **既存システム資料**: アーキテクチャ図、データフロー、ER図等
- **組織図・体制表**: 関係者の役割・連絡先
- **事前商談の文字起こし**: 顧客の課題認識・期待値の生の声
- **過去プロジェクトの成果物**: 類似案件があれば参照

### ② テキスト化

`scripts/convert-to-text.mjs` で受領資料を一括テキスト変換する。

```bash
node convert-to-text.mjs
```

- 対象: drive/ 配下の xlsx / pptx / pdf ファイル
- 出力: 同ディレクトリに .txt ファイルを生成
- 既に .txt がある場合はスキップ（--force で強制再変換）
- テキスト化済みファイルをClaude Codeに読み込ませることで、キックオフMDへの情報マッピングを効率化

### ③ キックオフMDへの流し込み

`templates/kickoff-format.md` の6セクション構造に沿って記述する。

#### 各セクションへの情報マッピング

| セクション | 何を書くか | 主なインプット源 |
|-----------|-----------|-----------------|
| 1. プロジェクト概要 | 背景・目的・体制 | 提案書、契約書、組織図 |
| 2. 弊社理解 | 現状認識・ゴール | ヒアリングメモ、商談文字起こし、既存システム資料 |
| 3. 論点 | 判断が必要な事項 | 商談文字起こし、技術調査結果 |
| 4. アプローチ | フェーズ・実施事項・IPO | 提案書、類似案件の実績 |
| 5. マイルストーン | いつまでに何が完了するか | 契約書（期間）、顧客の制約 |
| 6. リスクと前提条件 | 計画の成立要件・起きうる問題 | 全インプットから抽出 |

#### 記述のコツ

- **セクション1（背景）**: 「Pain → Gain」のストーリーで書く。3〜5行で簡潔に
- **セクション2（弊社理解）**: これは分析結果ではない。「我々はこう理解していますが合っていますか？」の認識合わせ
- **セクション3（論点）**: 「答え」は書かない。「判断が必要であること」を共有する場
- **セクション4（IPO整理）**: アウトプットには形式（MD/PPTX/Excel等）を併記。実施事項=Backlog親課題、アウトプット=Backlog子課題として運用する前提で設計
- **セクション5（マイルストーン）**: 月次テーブルのみ。詳細スケジュールは別管理（二重更新の回避）
- **セクション6（リスク）**: 既知のリスクを承知の上で進めるという合意を取る

### ④ すり合わせ

- MDファイルを共有し、関係者と認識合わせ
- **セクション2「弊社理解」が最重要**: ここの認識がズレていると全体がズレる
- 質問は1回のやり取りでまとめて投げる（逐次質問は顧客負担が大きい）
- 修正はMD上で行い、版管理する

### ⑤ PPTX化

合意後、以下のいずれかのフローでPPTXにする:

**フローA: Notion経由**
1. `scripts/create-kickoff-notion.mjs` でNotionページを作成
2. Notionで最終確認
3. NotionからPPTXテンプレートに転記

**フローB: 直接PPTX作成**
1. キックオフPPTXテンプレートにMDの内容を転記
2. 体制表・スケジュールはpptxgenjsで自動生成も可

## 別管理にするもの（キックオフMDには含めない）

| 資料 | 管理場所 | 備考 |
|------|---------|------|
| ファクトベース（事実一覧） | FACTS.md | 出典付きの確認済み事実 |
| 詳細スケジュール | スケジュールPPTX | キックオフMDからはリンク参照のみ |
| 未確定事項・次のアクション | Backlogチケット + 定例 | 流動的な情報のため別管理 |
| 詳細As-Is / To-Be分析 | PJ成果物として別途 | PJ進行中に作成 |

## PJT-YPでの教訓

1. **論点とIPOの紐づけが重要**: 04節のテーブルに「対応論点」列を設け、03節の論点番号（1-1, 2-3等）と紐づけることで抜け漏れを防止できた
2. **スコープ外の明示**: 「やらないこと」を明確に書くことでスコープクリープを防げた
3. **ミッション文はキャッチーに**: 「営業がスマホ1つで業務情報を確認できる世界」のような具体的なゴール状態を1文で書くと、全員の認識が揃いやすい
4. **テーマ間依存関係の明示**: テーマ2→テーマ1→テーマ3の依存関係を図示したことで優先順位の合意が取りやすかった
5. **IPO整理がスコープ管理の要**: 「子課題（アウトプット）を全て完了すれば親課題がクローズする」設計により、納品物の定義が明確になった
6. **弊社理解セクションは慎重に**: 確信度が低い情報は「〜と推察」等を付記し、事実と推論を混同しない
