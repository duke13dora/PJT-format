# 定例後対応 引き継ぎガイド

> 定例後のBacklog/資料更新をClaude Codeで実施する手順。

---

## 概要

定例の文字起こしを受け取り、以下を実施する:
1. 決定事項・議事メモ・ネクストの抽出
2. 決定事項シート(Excel)・議事メモ(docx)・DECISIONS.mdの更新
3. ネクストの洗い出し → Backlog反映

---

## インプット

- 定例の文字起こしテキスト（ユーザーから送付）

---

## ステップ1: 抽出

文字起こしから以下を抽出してチャットに一覧提示する:

- **決定事項**: 定例で合意・確定した内容（番号付き）
- **議事メモ**: 議論の要点サマリ
- **ネクスト**: 次のアクション一覧

### 文字起こし読み取りの注意事項

- **話者帰属は信頼しない**: マイク共有の場合あり。実際の話者を文脈から判断
- **日本の祝日は必ず自分で確認する**: 文字起こしの「来週祝日」等を鵜呑みにしない
- **曖昧語禁止**: 「概ね順調」→「5タスク中4タスク完了」等、必ず具体的に

---

## ステップ2: 決定事項・議事メモの記録

確認不要で即実行。完了後にチャットで報告し、ユーザーが確認→修正あれば対応。

### 2-1. DECISIONS.md更新

- 通し番号を前回から継続
- フォーマット:

```
| # | 決定日 | 決定者 | 概要 | 詳細 | 備考 |
```

### 2-2. Excel決定事項シート更新

- ツール: **adm-zip XML直接編集**（ExcelJS/SheetJS禁止）
- 手順: sharedStrings.xmlに文字列追加 → worksheets/sheetN.xmlの行追加 → writeZip
- 書式: 既存行のスタイルID・ht値をそのままコピーして値だけ変更
- 詳細: `guides/02-excel-editing.md` 参照

### 2-3. docx議事メモ更新

- ツール: **JSZip XML直接編集**
- 構成: 決定事項 → ネクスト → 議事メモ の3セクション
- 書式: 既存エントリの段落スタイル（w:pPr）をコピーして内容だけ変更
- 詳細: `guides/07-docx-editing.md` 参照

---

## ステップ3: ネクストの洗い出しとBacklog反映

**客先に通知が行く可能性がある。必ずユーザー確認を経てから実行。**

### フォーマット

| 項目 | 内容 |
|------|------|
| 内容 | やること概要 |
| 担当者 | Backlog上の担当者（assigneeId） |
| メンション | コメントで通知する宛先（notifiedUserId） |
| 期限 | いつまでか |
| 紐づけ先 | 既存チケットの子課題 or 新規チケット |
| 対応方法 | 子課題作成 / コメント追記 / 新規チケット / 既存チケット更新 |

---

## 絶対厳守ルール

1. **客先に影響する操作は全て事前確認必須**
2. **チケット起票時**: カテゴリ・マイルストーンを必ず設定。起票前に既存チケットを検索（重複防止）
3. **コメント**: プレーンテキストで書く（マークダウン記法禁止）

## ワークフローまとめ

```
文字起こし受領
  │
  ▼
決定事項・議事メモ・ネクスト抽出（チャットで一覧提示）
  │
  ├─ 決定事項・議事メモ → 確認不要で即実行
  │    ├─ DECISIONS.md 更新
  │    ├─ Excel 決定事項シート更新（adm-zip）
  │    └─ docx 議事メモ更新（JSZip）
  │    → 完了報告 → ユーザー確認 → 修正あれば対応
  │
  └─ ネクスト → フォーマットで一覧出力 → ユーザー確認
                                          │
                                          ▼
                                   Backlog操作実行
```
