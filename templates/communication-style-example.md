> このファイルは `MEMORY.md` の別ファイルとして配置し、プロジェクト固有にカスタマイズして使う。
> 配置先: `C:\Users\{USER}\.claude\projects\{PROJECT_DIR}\memory\communication-style.md`

## Backlog 対外コメント（顧客向け）

### 基本トーン

- **丁寧だが簡潔**: 冗長な前置きなし。要点をすぐ伝える
- **謙譲語ベース**: 一貫して「いたしました」「いたします」を使用
- **プレーンテキスト**: マークダウン記法禁止（`**太字**` → `*` がそのまま表示される）

### 構造記号

| 記号 | 用途 | 例 |
|------|------|-----|
| ■ | 大項目の見出し | `■ 対応内容` |
| ※ | 補足・経緯・意図 | `※既存の設定を活かす方針で対応しました` |
| → | アクション・結果 | `→資料を格納しました` |
| - | 箇条書き | `- 追加コストが不要` |

### 冒頭謝意の使い分け

| 状況 | 表現 |
|------|------|
| 指摘を受けた | 「ご指摘ありがとうございます。」 |
| 意見・情報提供 | 「ご意見ありがとうございます。」 |
| ミスの指摘 | 「〜ができておらず失礼いたしました。今後は〜するようにいたします。」 |
| 確認依頼 | 「ご確認をお願いいたします。」 |

### 相手の意見への応答パターン

1. 相手の発言を自分の言葉で箇条書き要約
2. 「〜旨理解いたしました」で締める

### NG表現一覧

| NG表現 | 正しい表現 |
|--------|-----------|
| 「させていただきます」 | 「いたします」 |
| 「ご査収ください」 | 「ご確認をお願いいたします」 |
| 「何卒」「幸いです」 | 使わない（削除） |
| 「お手数をおかけしますが」 | 「お手数ですが」または省略 |
| 長い前置き | 冒頭1行の謝意のあとすぐ本題 |
| `**太字**` 等マークダウン | ■や※で構造化 |

## Slack 社内メッセージ（チームメンバー向け）

### 基本トーン

- **カジュアル寄り**: 「あざます」「合ってますよね？」OK
- **簡潔直球**: 前置きなし、結論ファースト
- **アウトライン形式**: 散文禁止

### 構成パターン（複数件連絡時）

```
@{相手の名前}
{件名}、{N}件連絡です
ーーーーー

PJ-XXX（概要）：{URL}
{状況説明}
→{アクション依頼}（チケット上で

PJ-YYY（概要）：{URL}
{状況説明・認識確認}
→{確認依頼}
- {確認ポイント1}
- {確認ポイント2}
```

### 特徴

- **チケット番号（概要）：URL** を1行にまとめる（■は使わない）
- **→** でアクション依頼を示す
- **ーーーーー** でヘッダーと本文を区切る
- **クロージングなし**: 最後の箇条書きで終わる

## 共通禁止事項

- Backlogコメントにマークダウン記法を使わない
- AI感のある過剰な敬語を使わない
- 先方への呼称は「様」（Backlog）、社内では「さん」（Slack）
